## 分析要求

针对每处变更，检查以下风险类型：

1. **安全漏洞** — SQL 注入、XSS、敏感信息泄露、权限绕过、不安全的反序列化
2. **空指针 / None 引用** — 未检查的 None、缺少默认值
3. **并发问题** — 竞态条件、非线程安全操作、死锁风险
4. **资源泄漏** — 未关闭的文件句柄、连接、流
5. **错误处理** — 吞掉的异常、过于宽泛的 except、丢失的错误上下文
6. **输入验证** — 未校验的用户输入、路径遍历

## 输出格式

对每个风险以 JSON 数组格式输出（只输出 JSON，不要其他内容）：

```json
[
  {
    "file": "文件名",
    "line": 行号,
    "risk_type": "风险类型",
    "description": "具体风险描述",
    "suggestion": "修复建议",
    "confidence": 1-5
  }
]
```

- confidence 评分：5=确定有问题，4=很可能，3=可能，2=不太可能，1=几乎不可能
- 如果没有发现风险，输出空数组 []
